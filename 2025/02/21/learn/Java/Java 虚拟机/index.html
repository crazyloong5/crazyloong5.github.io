Java虚拟机

内存

内存分布

程序计数器

Java虚拟机中程序计数器与计算机组成原理（Cpu）里的程序计数器效果相同，按照一定的顺序记录当期执行的字节码的行号，并根据字节码进行跳转、循环等操作。

因为Java实现高并发是通过不停的切换线程实现，所以每条线程都会拥有自己独立的程序计数器，不然就会造成程序执行混乱。这类内存可以称作”线程私有“的内存。

如果当前执行的是Java方法，那么程序计数器记录的是当前执行方法的地址；如果当前执行的是非Java方法（Native方法），那么程序计数器为空。

tip：线程私有的内存生命周期与线程相同

虚拟机栈

虚拟机栈即Java虚拟机的内部栈，与程序计数器一样，它也是“线程私有”。

每当Java方法执行时，Java虚拟机都会创建一个栈帧，这个栈帧存储了编译期可知的各种局部变量表、操作数栈、动态链接、方法出口等信息。其中局部变量表存储了编译时可知的基础数据类型（bite, int, long, flout, double, char, boolean）、引用类型(reference)和returnAddress（指向一条方法字节码的地址）。其中64位的long和double会占2个局部变量空间，其余的数据类型占1个。局部变量表所需的内存在编译期就会分配完成，在方法运行时不再变动，故当进入一个方法时，这个方法在帧中分配多大的局部变量空间是完全确定的。

一个Java方法从调入到执行完成的过程就是栈帧在虚拟机栈出栈到入栈的过程。

可能出现异常：StackOverflowError、OutOfMemoryError

本地方法栈

本地方法栈与虚拟机栈相似，唯一的不同点是虚拟机栈为java方法服务，而本地方法栈为虚拟机使用到的Native方法服务。

tip：虚拟机对本地方法栈中方法使用的语言、方式、数据结构都没有强制规定。

可能出现异常：StackOverflowError、OutOfMemoryError

Java堆

Java堆（被所有线程共享的一块内存区域，虚拟机启动时创建），存放对象实例、数组。

因为时垃圾收集器管理的主要区域，所以很多活时候也叫做“GC堆”。

Java堆放在内存不连续的内存空间，只要逻辑上连续即可。

Java堆既可以实现成固定大小的，也可以实现成可扩展的。

可能出现异常：OutOfMemoryError

方法区（永久代）

与java堆（堆的一个逻辑部分）相似但又不完全一样（Non-Heap），它也是线程共享的内存区域，存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码等数据。

方法区放在内存不连续的内存空间，只要逻辑上连续即可。

方法区既可以实现成固定大小的，也可以实现成可扩展的。

方法区可以选择不是想垃圾收集。

内存回收主要时针对常量池的回收和对类型的卸载。

可能出现异常：OutOfMemoryError

运行时常量

方法区的一部分，存放Class文件编译期生成的各种自变量、符号引用和翻译出来的直接引用。

不同提供商实现的虚拟机可以按照自己的需要来实现这个内存区域。

具备动态性，运行时可能将新的常量放入池中，如String类的intern()方法。

可能出现异常：OutOfMemoryError

直接内存

不是java虚拟机运行时数据区的一部分，也不是java虚拟机规范中定义的内存区域，本质是通过系统提供的malloc等接口分配内存区域。

Java中一种基于操作系统内存的区域，直接分配在操作系统内存中，不受jvm垃圾回收机制的管理，因此分配和释放都需要程序员手动管理。

JDK1.4中新加入NIO类，引人了一种基于通道与缓冲区的I/O方式，它可以使用Native函数库直接分配堆外内存，然后通过一个存储在Java堆中的DirectByteBuffer对象作为这块内存的引用进行操作。

受到本机总内存大小以及处理器寻址空间的限制。

可能出现异常：OutOfMemoryError
