1. 用静态(工厂)方法替代构造器

例子

优势

  1. 静态(工厂)方法有名称
  2. 不必在每次调用它们的时候都创建一个新的对象
  3. 它们可以返回原返回类型的任何子类型的对象
  4. 所返回的对象的类可以随着每次调用而发生变化，这取决于静态工厂方法的参数值
  5. 方法返回的对象所属的类，在编写包含该静态工厂方法的类时可以不存在

劣势

  1. 类如果不含公有的或者受保护的构造器，就不能被子类化。
  2. 程序员很难发现它们。

2. 遇到多个构造器参数时要考虑使用构建器

例子

优点

  1. 适用于多参数组合，参数越多越适合。
  2. 当你需要的参数组合类型/变化很多（final / static），便于我们创建对象，且可以内部提供安全校验。
  3. 通过builder创建对象十分灵活。

缺点

  1. 需要先创建builder，浪费性能。
  2. 若是一个已经使用构造器铺开的类，想要使用builder进行重构的维护成本很高。

3. 用私有构造器或者枚举类型强化 Singleton 属性

例子

优点

  1. API很清楚地表明了这个类是一个 Singleton：公有的静态域是 final的，所以该域总是包含相同的对象引用。
  2. 它更简单。
  3. 它更灵活，在不改变其API的前提下，我们可以改变该类是否应该为Singleton的想法。
  4. 如果应用程序需要，可以编写一个泛型Singleton工厂。
  5. 可以通过方法引用作为提供者。
